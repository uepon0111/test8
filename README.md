# プロセカ リザルト管理

プロジェクトセカイのプレイ結果（リザルト画面のスクリーンショット）をOCRで自動解析し、Google Drive に保存・一覧管理する GitHub Pages 用Webアプリです。

## 今回のアップデート内容

- **EASY / NORMAL 難易度に対応**（EASY・NORMAL・HARD・EXPERT・MASTER・APPENDの6難易度）
- **PERFECT数・GREAT数・コンボ数の計測・保存に対応**（既存のGOOD/BAD/MISS読み取り範囲を拡張し、コンボ数用の読み取り範囲を新規追加）
- **Google Drive の保存方式を効率化**（詳細は下記「Driveの保存構造」を参照）
- **設定ページ（`settings.html`）を新設**し、OCRの読み取り範囲をサンプル画像を見ながらGUIで調整可能に
  - 矩形のドラッグ＆リサイズ、および数値入力（X/Y/W/H%）の両方で範囲を指定可能（相互にリアルタイム反映）
  - 読み取り範囲は「機種プロファイル」として複数保存可能
- **アップロード時に画像の解像度・アスペクト比から機種プロファイルを自動選択**（手動切り替えも可能）
- **自己ベスト更新時にページ内でトースト通知**
- **「自己ベストのみ表示」トグル**を追加
- **並び替え・絞り込み中のロード状況を視覚的に表示**（進捗バー）
- **並び替えの優先順位を指定どおりに実装**（各モードの主キーは昇順・降順を切替可能）
  - 名前順　　　: 名前 → 楽曲難易度 → ミス数 → 追加日
  - 楽曲レベル順: レベル → 楽曲難易度 → 名前 → ミス数 → 追加日
  - ミス数順　　: ミス数 → 楽曲レベル → 楽曲難易度 → 名前 → 追加日
  - 追加日順　　: 追加日のみ
- **機能・役割ごとにファイルを分割**（下記「ファイル構成」を参照）

既存のUI・処理（見た目のデザイン、FC判定の基準、曲名マッチング処理など）は、上記の変更に直接関係しない部分はそのまま維持しています。

## Driveの保存構造（変更点）

**旧方式**: `プロセカリザルト/FC/{レベル}{難易度} {曲名}/FC(-{ミス数})`  
楽曲ごとにフォルダを検索・作成する必要があり、Drive APIの呼び出し回数が楽曲数に比例して増える構造でした。

**新方式**: `プロセカリザルト/results/` フォルダ1つに全リザルト画像をフラットに格納します。

- ファイル名: `{レベル}{難易度コード}_{曲名}_{タイムスタンプ}.拡張子`（例: `28EX_サンプル楽曲_20260708-153045.jpg`）
  - Driveで直接見たときに内容が分かるように付与していますが、アプリ自体はここから値をパースすることを前提にしていません。
- 実際の数値データ（PERFECT/GREAT/GOOD/BAD/MISS/コンボ数/難易度/レベル/楽曲IDなど）は、Driveファイルの **`appProperties`**（カスタムメタデータ）に保存しています。
- 「追加日」はDrive標準の `createdTime` をそのまま利用しています（自前でタイムスタンプを管理する必要がありません）。

この方式により、一覧取得は「resultsフォルダの検索1回＋ファイル一覧取得」のみで完了し、アップロード時も楽曲ごとのフォルダ検索が不要になります。

> **ご注意**: 保存構造を変更したため、**旧方式（`FC`フォルダ配下）に保存されていた過去のリザルトは、新バージョンの一覧には自動的には表示されません。** ご要望に含まれていた「Drive内のファイル名・フォルダ名の命名方法の変更」の一環による仕様です。過去データを引き続き利用したい場合は、お手数ですが該当の画像を再アップロードしてください。

## 機種プロファイル（読み取り範囲の設定）について

- 設定は各ブラウザの `localStorage` に保存されます（Google Driveには保存されません）。そのため、別のブラウザ・端末からアクセスした場合は、初回は再度設定が必要です。
- サンプル画像そのものはアップロード・保存されません。範囲調整のためにその場で表示するだけで、ページを離れると破棄されます。
- 初期状態では「デフォルト」プロファイルが1つ用意されており、これまでの読み取り範囲に近い値が入っています。読み取り精度が低い場合は、設定ページからお使いの端末のスクリーンショットを使って範囲を調整してください。

## 動作確認について

このサンドボックス環境からは Google API（OAuth / Drive）への実通信ができないため、実際のログイン・アップロード・Drive保存については確認できていません。以下は確認済みです。

- 全JSファイルの構文チェック（`node --check`）
- 並び替え・絞り込み・自己ベスト判定・機種プロファイル自動選択・Driveファイル名/メタデータの相互変換・OCRの難易度判定と判定内訳パース処理について、Node.js上でのユニットテスト（99件、すべて成功）
- HTML内の要素IDとJSからの参照の突合、`onclick`等から呼び出す関数が実際に存在するかの突合（ground-truth実行による検証）
- HTMLタグの対応関係、CSSクラスの定義漏れの確認

実際にGitHub Pagesへデプロイした後、初回ログイン・アップロード・設定ページでの範囲調整を一通りお試しいただくことをおすすめします。
